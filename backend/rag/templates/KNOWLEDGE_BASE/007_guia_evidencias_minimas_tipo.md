# Evidencias Mínimas por Tipo de Servicio

## Objetivo

Establecer la documentación mínima requerida para cada tipo de servicio contratado,
bajo el prisma de LISR 27 + CFF 69-B + CFF 5-A.

Este documento es usado por todos los agentes para:
- A2_PMO: Validar completitud de expediente
- A3_FISCAL: Evaluar deducibilidad
- A7_DEFENSA: Preparar defense file

---

## 1. CONSULTORÍA ESTRATÉGICA / MANAGEMENT

### Descripción
Servicios de asesoría en estrategia, organización, transformación de negocios.

### Evidencia mínima requerida

| Tipo | Documento | Propósito | Obligatorio |
|------|-----------|-----------|-------------|
| Contractual | Contrato marco de servicios | Establecer alcance y condiciones | SÍ |
| Contractual | SOW por proyecto | Detallar entregables específicos | SÍ |
| Entregable | Diagnóstico inicial | Evidencia de trabajo realizado | SÍ |
| Entregable | Reporte final / recomendaciones | Producto tangible del servicio | SÍ |
| Comunicación | Minutas de reuniones | Evidencia de interacción | SÍ |
| Comunicación | Correos de avance | Trazabilidad de servicio | RECOMENDADO |
| Aprobación | Acta de aceptación firmada | Conformidad del cliente | SÍ |
| Razón de negocios | Memo de justificación del proyecto | Por qué se contrató | SÍ |

### Red flags específicos
- Contrato sin alcance detallado
- Sin entregables documentados
- Facturación sin correspondencia con SOW

### Score base: +15 puntos si todo completo

## 2. SERVICIOS DE TECNOLOGÍA / SOFTWARE / SAAS

### Descripción
Desarrollo de software, licencias, SaaS, mantenimiento de sistemas, TI.

### Evidencia mínima requerida

| Tipo | Documento | Propósito | Obligatorio |
|------|-----------|-----------|-------------|
| Contractual | Contrato de licencia o desarrollo | Establecer derechos y obligaciones | SÍ |
| Contractual | SLA (Service Level Agreement) | Definir niveles de servicio | RECOMENDADO |
| Entregable | Código fuente / acceso a plataforma | Evidencia de producto | SÍ |
| Entregable | Documentación técnica | Evidencia de desarrollo | SEGÚN CASO |
| Uso | Logs de acceso / uso de sistema | Evidencia de utilización | SÍ |
| Uso | Tickets de soporte atendidos | Evidencia de mantenimiento | SÍ si es soporte |
| Comunicación | Correos de entrega de versiones | Trazabilidad | RECOMENDADO |
| Aprobación | Acta de aceptación / Go-live | Conformidad funcional | SÍ |

### Red flags específicos
- Software sin evidencia de uso
- Licencias sin acceso verificable
- Desarrollo sin entregables de código/docs

### Score base: +12 puntos si todo completo

## 5. SERVICIOS INTRAGRUPO / MANAGEMENT FEE

### Descripción
Servicios entre empresas relacionadas: administrativos, de gestión, soporte.

### Evidencia mínima requerida (CRÍTICA por riesgo de precios de transferencia)

| Tipo | Documento | Propósito | Obligatorio |
|------|-----------|-----------|-------------|
| Contractual | Contrato intercompañía | Establecer servicios y precios | SÍ |
| Precios | Estudio de precios de transferencia | Justificar arm's length | SÍ |
| Entregable | Reportes de servicios prestados | Evidencia de actividad | SÍ |
| Metodología | Bases de prorrateo / cost plus | Justificar cálculo | SÍ |
| Comunicación | Evidencia de interacción entre empresas | Trazabilidad | SÍ |
| Aprobación | Aceptación de servicios por periodo | Conformidad | SÍ |
| Razón negocios | Justificación de esquema intragrupo | Por qué no directo | SÍ |

### Red flags específicos
- Sin estudio de precios de transferencia
- Management fee sin detalle de servicios incluidos
- Prorrateos sin metodología documentada
- Montos que no pasan test arm's length

### Score base: +18 puntos si todo completo (peso extra por riesgo)

## USO EN SCORING DE AGENTES

Los agentes deben usar esta guía para:

1. **A2_PMO**: Verificar que el expediente tenga todos los documentos
2. **A3_FISCAL**: Ajustar score de deducibilidad según completitud
3. **A7_DEFENSA**: Identificar gaps antes de preparar defense file
4. **A8_AUDITOR**: Verificar cumplimiento de evidencia mínima

### Cálculo de ajuste al score

```
Score ajustado = Score base del tipo + (Evidencias completas / Evidencias requeridas) * 20

Si evidencias < 50%: Penalización de -15 puntos
Si proveedor en 69-B: Penalización de -30 puntos
```
