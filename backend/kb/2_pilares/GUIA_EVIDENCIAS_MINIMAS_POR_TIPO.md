# [GUIA] Evidencias Minimas por Tipo de Servicio

## Objetivo

Establecer la documentacion minima requerida para cada tipo de servicio contratado,
bajo el prisma de LISR 27 + CFF 69-B + CFF 5-A.

Este documento es usado por todos los agentes para:
- A2_PMO: Validar completitud de expediente
- A3_FISCAL: Evaluar deducibilidad
- A7_DEFENSA: Preparar defense file

---

## 1. CONSULTORIA ESTRATEGICA / MANAGEMENT

### Descripcion
Servicios de asesoria en estrategia, organizacion, transformacion de negocios.

### Evidencia minima requerida

| Tipo | Documento | Proposito | Obligatorio |
|------|-----------|-----------|-------------|
| Contractual | Contrato marco de servicios | Establecer alcance y condiciones | SI |
| Contractual | SOW por proyecto | Detallar entregables especificos | SI |
| Entregable | Diagnostico inicial | Evidencia de trabajo realizado | SI |
| Entregable | Reporte final / recomendaciones | Producto tangible del servicio | SI |
| Comunicacion | Minutas de reuniones | Evidencia de interaccion | SI |
| Comunicacion | Correos de avance | Trazabilidad de servicio | RECOMENDADO |
| Aprobacion | Acta de aceptacion firmada | Conformidad del cliente | SI |
| Razon de negocios | Memo de justificacion del proyecto | Por que se contrato | SI |

### Red flags especificos
- Contrato sin alcance detallado
- Sin entregables documentados
- Facturacion sin correspondencia con SOW

### Score base: +15 puntos si todo completo

## 2. SERVICIOS DE TECNOLOGIA / SOFTWARE / SAAS

### Descripcion
Desarrollo de software, licencias, SaaS, mantenimiento de sistemas, TI.

### Evidencia minima requerida

| Tipo | Documento | Proposito | Obligatorio |
|------|-----------|-----------|-------------|
| Contractual | Contrato de licencia o desarrollo | Establecer derechos y obligaciones | SI |
| Contractual | SLA (Service Level Agreement) | Definir niveles de servicio | RECOMENDADO |
| Entregable | Codigo fuente / acceso a plataforma | Evidencia de producto | SI |
| Entregable | Documentacion tecnica | Evidencia de desarrollo | SEGUN CASO |
| Uso | Logs de acceso / uso de sistema | Evidencia de utilizacion | SI |
| Uso | Tickets de soporte atendidos | Evidencia de mantenimiento | SI si es soporte |
| Comunicacion | Correos de entrega de versiones | Trazabilidad | RECOMENDADO |
| Aprobacion | Acta de aceptacion / Go-live | Conformidad funcional | SI |

### Red flags especificos
- Software sin evidencia de uso
- Licencias sin acceso verificable
- Desarrollo sin entregables de codigo/docs

### Score base: +12 puntos si todo completo

## 5. SERVICIOS INTRAGRUPO / MANAGEMENT FEE

### Descripcion
Servicios entre empresas relacionadas: administrativos, de gestion, soporte.

### Evidencia minima requerida (CRITICA por riesgo de precios de transferencia)

| Tipo | Documento | Proposito | Obligatorio |
|------|-----------|-----------|-------------|
| Contractual | Contrato intercompania | Establecer servicios y precios | SI |
| Precios | Estudio de precios de transferencia | Justificar arm's length | SI |
| Entregable | Reportes de servicios prestados | Evidencia de actividad | SI |
| Metodologia | Bases de prorrateo / cost plus | Justificar calculo | SI |
| Comunicacion | Evidencia de interaccion entre empresas | Trazabilidad | SI |
| Aprobacion | Aceptacion de servicios por periodo | Conformidad | SI |
| Razon negocios | Justificacion de esquema intragrupo | Por que no directo | SI |

### Red flags especificos
- Sin estudio de precios de transferencia
- Management fee sin detalle de servicios incluidos
- Prorrateos sin metodologia documentada
- Montos que no pasan test arm's length

### Score base: +18 puntos si todo completo (peso extra por riesgo)

## USO EN SCORING DE AGENTES

Los agentes deben usar esta guia para:

1. **A2_PMO**: Verificar que el expediente tenga todos los documentos
2. **A3_FISCAL**: Ajustar score de deducibilidad segun completitud
3. **A7_DEFENSA**: Identificar gaps antes de preparar defense file
4. **A8_AUDITOR**: Verificar cumplimiento de evidencia minima

### Calculo de ajuste al score

```
Score ajustado = Score base del tipo + (Evidencias completas / Evidencias requeridas) * 20

Si evidencias < 50%: Penalizacion de -15 puntos
Si proveedor en 69-B: Penalizacion de -30 puntos
```
